# `.agents/`

This directory holds AI coding-agent instructions for this project: slash
commands, skill docs, the MCP server config, and the Docker / environment
hints for cloud runtimes (Cursor Cloud, Claude Code).

**You (the agent) do not need to install or maintain this directory.**
Most files in here are auto-installed from the
`@yottagraph-app/aether-instructions` npm package at provisioning time â€”
they are **not** a `package.json` dependency, are **not** in
`node_modules`, and are **not** something you should add to imports.

If you arrived here from `ls`-ing the tree and were wondering "is this an
npm dependency I'm missing?", the answer is no â€” the package's install
script (called by `node init-project.js`) copies its payload directly
into this directory. The mechanism is unusual but intentional: it keeps
agent instructions decoupled from the app's runtime deps.

## What's where

| Path                            | Source                         | What it is                                                                  |
| ------------------------------- | ------------------------------ | --------------------------------------------------------------------------- |
| `commands/`                     | npm package (auto-installed)   | Slash commands like `/build_my_app`, `/deploy_agent`                        |
| `skills/`                       | npm package (auto-installed)   | Skill docs you read on demand â€” read `skills/aether/SKILL.md` first         |
| `Dockerfile`                    | aether template                | Build hints for Cursor Cloud sessions                                       |
| `environment.json`              | aether template                | Cursor Cloud "Dev Server" + install-command config                          |
| `mcp.json`                      | generated by `init-project.js` | MCP server URLs (Lovelace MCP servers via the Broadchurch Portal proxy)     |
| `.aether-instructions-manifest` | written by `init-project.js`   | Lists every file the npm package installed (used by `/update_instructions`) |
| `.aether-instructions-version`  | written by `init-project.js`   | Installed package version (for diagnostics)                                 |

`.cursor/`, `.claude/`, and `.mcp.json` at the repo root are symlinks
into this directory, so Cursor and Claude Code both read the same files
without duplication.

## Refreshing

To pull the latest commands + skills from a new package release, run the
`/update_instructions` slash command. It re-downloads the npm package,
removes only the files listed in `.aether-instructions-manifest`, and
re-installs them. Anything else in `.agents/` (your tenant-specific
`Dockerfile`, `environment.json`, `mcp.json`) is left alone.

## Don't edit installed files in place

The files listed in `.aether-instructions-manifest` get overwritten on
the next `/update_instructions`. If you need to override a skill or
command, see `skills/aether/instructions_warning.md` for the
copy-and-edit-the-copy pattern.

## Where the package lives

The npm package source is in the upstream development repository â€” you
do not need to know which one, and it's intentionally not referenced
elsewhere in this project. If the `/update_instructions` command can't
reach it (e.g. offline session, npmjs outage), the existing `.agents/`
contents keep working â€” they're plain files, not a live dependency.
